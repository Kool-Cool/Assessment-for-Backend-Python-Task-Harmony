## System Design Question
Scale
Use a queue‑based architecture (Kafka/SQS) with containerized workers pulling emails for processing.

Apply extraction via cost‑efficient multilingual models or translation pipelines, store results in NoSQL (MongoDB/DynamoDB).

Autoscale workers to handle bursts, batch requests to keep compute costs low, staying within $500/month while meeting 5‑minute SLA.

Monitoring
Track accuracy daily with sampled emails, log field‑level and record‑level metrics in dashboards (Grafana/Prometheus).

Set alerts if accuracy dips below thresholds (e.g., 85%).

Investigate by isolating degraded fields, checking for data drift (new formats/languages), and reviewing logs for parsing/prompt errors.

Multilingual
Add multilingual support: either use a multilingual LLM or translate Mandarin/Hindi emails into English before extraction.

Build separate test sets for Mandarin, Hindi, and English, measure accuracy per language group.
